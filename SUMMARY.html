1. Overview

The objective was to transmit a continuous UDP media stream across an unreliable relay, maintaining a deadline miss rate <1% within a 2x bandwidth constraint. Our protocol solves packet loss, duplication, reordering, and jitter using overlapping Forward Error Correction (FEC), a thread-safe circular jitter buffer, and precise deadline-driven scheduling.

2. System Architecture

Harness Source → Sender → Custom UDP Protocol → Relay → Receiver → Harness Player

The sender processes one audio frame every 20ms, transforming raw input into two distinct packet types: Data Packets (original payload) and Parity Packets (sliding XOR window). This allows the receiver to reconstruct missing data locally without network-latency-heavy retransmissions.

3. Design Choices & System-Level Details

  • Buffer Ownership & Concurrency: We utilized C11 stdatomic.h to manage the circular buffer. The receive thread owns the write-access, and the playback thread owns the read-access. Atomic operations ensure memory visibility without the overhead of heavy pthread_mutex locks.
  • Byte Order: All sequence numbers and parity base indices are serialized using Network Byte Order (Big-Endian) via htonl()/ntohl(), ensuring platform-agnostic packet parsing.
  • Non-blocking Safety: The receive loop is strictly non-blocking. It handles incoming UDP datagrams in a continuous loop, pushing parity blocks to the recovery engine instantly. The system never blocks on I/O, ensuring it can process bursts of packets during jitter spikes.
  • Timing Strategy: To eliminate OS scheduler jitter, we utilize a two-stage scheduler: usleep() until 2ms before the deadline, followed by a tight lock-free busy-spin that monitors the atomic received flag until 20µs before playout.

4. Network Handling Strategy

Network Issue Handling Strategy
Packet Loss Overlapping sliding-window XOR parity recovery.
Packet Reordering Sequence-indexed circular buffer (O(1) insertion).
Duplicate Packets Ignored using atomic received state flags.
Jitter Variation Receiver-side playout jitter buffer.

5. Constraints & Limitations

  • Buffer Size: The jitter buffer is sized at 65,536 slots to ensure it can survive the extreme burst-loss and latency scenarios presented in Profile B.
  • FEC Logic: Recovery succeeds when exactly one packet inside a protected sliding window is missing. Burst losses spanning the entire window cannot be reconstructed.
  • Dynamic Adaptation: The playback delay is currently fixed via CLI configuration; while optimal for the given profiles, a production system would adapt this dynamically based on RFC 3550 variance estimations.

6. Conclusion

This implementation provides a lightweight, robust real-time transport protocol. By combining lock-free atomics with overlapping XOR FEC, the design provides high resilience against network instability while strictly adhering to the 2x bandwidth overhead and <1% miss rate constraints.
